AEC camera startup parameters

The Parameter Editor has startup parameters that can be adjusted to achieve the fastest startup AEC convergence speed when the camera starts from the device UI.

Parameter Description Tuning
First frame
firstFrameStepSizeDarkMax Increase this to allow a larger step size when converging from dark to bright. A larger value may cause overshoot. Range: [0, 200]
maxAdjFactor When converging from bright to dark, set this factor to boost the adjustment step size for the first frame. A larger value will converge faster, but may overshoot. Range: [1.5, 3.0]
stepSizeBrightAdjRange Set this range to control when to apply maxAdjFactor. Set a lower value to apply the factor only when AEC is close to the target. Below the specified range, a factor of 1.0 is applied; above this range, maxAdjFactor is applied. In between this range, a linear interpolation within [1.0, maxAdjFactor] is applied. Range: [20, 80]
Convergence
tolMultUnitStepSize Similar to regular convergence tuning parameter, where increasing this parameter causes leniency on when the exposure is within an acceptable range. Range: [2, 6]
frameSkipInit Set this to a positive value to skip frames before starting to converge. Range: [0, 4]
frameSkip Set this value to the system delay between setting an exposure and receiving matching stats (typically, two frames are skipped). Range: [2, 3]
minSettledCount Increase this parameter value to make AEC wait longer before declaring itself settled. Range: [3, 5]
maxFrameConvTimeOut Decrease this parameter value to give less time for AEC to converge, taking the risk that AEC may not complete converging. Range: [10, 50]
Arbitration
expTableIDAggressive Set to the exposure table index for aggressive AEC. Range: all
expTableIDFastAEC Set to the exposure table index for fast AEC. Range: all
Generic
startExpIndex
  • AEC exposure index when camera starts from device UI.
  • When this parameter has a higher value, the AEC is biased toward a darker environment, such as indoor and lowlight.
  • When this parameter has a lower value, the AEC is biased toward a brighter environment such as outdoor.
  • Usually a bright glitch is more annoying than a dark glitch. Therefore, it is recommended to keep this value lower, between 150 and 250.
 Range: [0, 500]
startUpAggressiveness
  • Use this parameter to override the speed for AEC convergence at startup.
  • The value must not exceed 1 to avoid over/under shoot issues at startup.
  • Increase this value if you want AEC to converge faster at camera startup.
  • Decrease if overshoot/undershoot occurs at startup.
 Range: [0, 1]
