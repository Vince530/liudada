Collect and verify lens position data on golden modules

  1. Complete the actuator range test available at Live > Utilities > 3A Scan Test > CAF Scan Test.
    • If the graphs look as expected, then the OTP data provided by vendor is valid.

    • If the graphs do not look as expected, then there could be a problem with the OTP data that is causing the lens not to focus on either macro or infinity.

      • The best solution is to contact the vendor and verify if the OTP data they provided are correct or not.
      • The second solution is to add MARGIN over MACRO DAC and INFINITY DAC to take care of module and temperature variations. For additional assistance, contact the sensor driver team or CE engineer.
  2. Compare the lens position for different distances with the value in the lens position tuning values section of the AF Tuning tab.

  3. Enable logging, use the ISO 12233 chart, and run the full-sweep search for the following distances: 7 cm, 10 cm, 14 cm, 20 cm, 30 cm, 40 cm, 50 cm, 60 cm, and 120 cm.
  4. Record the lens position from the logs.
  5. Compare the value obtained from the full-sweep search with the values in the lens position tuning values section of the AF Tuning tab.
    • If the values match or if they are close, then there is no need to modify the value in header.
    • If the values do not match or if there is huge difference, then update the header with the values obtained from full-sweep test.
  6. In general, the lens position values calculated in Chromatix and the results from the full-sweep test will not match exactly, but they should be close. The values calculated in Chromatix are theoretical values based on input from the lens sensor specifications. There is a possibility to see variations in the modules. If there are some variations, then copy the values obtained from the full-sweep test. To account for module variation, repeat the full-sweep search using different modules and compare those results to the lens position tuning values in the AF Tuning tab.
