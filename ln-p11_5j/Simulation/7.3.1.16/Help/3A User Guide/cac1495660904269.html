AF and TAF need to be optimized for speed but CAF needs to be optimized for smoothness

Tune the step tables for TAF and CAF separately.

  1. Check the step tables (separate table for CAF and TAF) available in the af_single_t structure in the AF header. There are different sets for normal light and low light.
  2. To prioritize for fast AF speed, tune the step table to be more aggressive, however, there is the possibility of sacrificing accuracy in certain cases.
  3. To improve smoothness, reduce the step size from the macro entry first (last entry of the step table) and then scale the reduction of the step size for the other entries.
Note: Tune the step table to be more conservative (hence accuracy), however, there is the possibility of sacrificing overall AF speed and failure rate in certain cases.

It should be trivial to see the immediate effect of changing the step size. Changing the step size affects the contrast between the samples, which affects the strength of the algorithm thresholds for determining the FV curve trend and the ensuing AF decision. Although tuning values are available, it would be very hard to tune the thresholds individually because they are tightly related to each other. Because the defaults are set up by balancing different AF priorities (e.g., steps size, speed, and success rate), the thresholds should also be tuned to accommodate changes in the step table. For example, if you increase the step size, then reduce the thresholds. Correspondingly, if you decrease the step size, then increase the thresholds.
