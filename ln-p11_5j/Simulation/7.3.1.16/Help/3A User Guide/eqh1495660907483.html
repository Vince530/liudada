Autofocus concepts

Autofocus determines the best lens position to achieve focus and triggers the actuator to move the lens to that position. There are many hardware and software components involved in autofocus, but in general, the process is as follows:

  1. AF algorithms concurrently monitor hardware and software stats to determine if there is a scene change.
  2. If the algorithm detects a scene change, the algorithm triggers a search. If any depth-based algorithms are enabled, CAF is only called as a fallback.
  3. The coarse search of the algorithm determines next lens position.
  4. The actuator incrementally moves the lens to the next position.
  5. The fine search finds the final lens position.
  6. The actuator moves the lens to the final position.

The AF algorithms are contained within the hybrid AF (HAF) infrastructure. The infrastructure is extensible and by default implements three depth-based algorithms (i.e., PDAF, TOF, DCIAF), as well as Contrast AF (CAF). Each algorithm implements monitor and search functionality.

The infrastructure is designed to allow customers and third parties to introduce their own logic into the AF decision process. Using HAF, it is possible to integrate new algorithms, modify the behavior of existing algorithms, and even change the mixing logic when multiple algorithms are combined.
