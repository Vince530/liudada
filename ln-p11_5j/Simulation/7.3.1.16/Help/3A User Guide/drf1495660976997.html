Remove or reduce banding noise



  1. Verify with the sensor vendor and sensor driver engineer rhat the correct sensor configuration for pixel clock, line_length_pixel, and frame_length_line have been implemented into the sensor driver. An incorrect configuration of these settings causes banding.
  2. Dump the kernel log and read I2C for these register settings to verify that the sensor driver configuration settings are correct.
  3. Capture an image with the test condition to verify the results. If the issue persists, capture a test image with 3A EXIF information and AEC log messages.
  4. In the 3A Debug dialog box, click the Data tab and select AEC Debug Info > Gain and Line Count Information for the test image.
  5. Check the AE Snapshot Exposure Time. If the exposure time is smaller than 1/120 sec in a 60 Hz condition or 1/100 sec in a 50 Hz condition, due to physical limitations, the banding issue occurs and cannot be fixed by ISP.
  6. Set the exposure time longer than 1/100 sec (50 Hz condition) or 1/120 sec (60 Hz condition) by tuning the exposure table.
  7. Capture an image with the test condition to verify the results. If the issue persists, check for power-related noise.
    Usually power noise is most obvious when using a large analog gain or during lens movement. Power noise may be due to:
    • A sensor analog power source
    • VCM power (especially if it uses PWM)
    • Bad or insufficient grounding (ground loop)
    Work with your hardware engineers to solve the power noise issue.
